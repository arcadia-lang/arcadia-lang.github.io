# <x-trans>Gender</x-trans>

<x-trans>Arcadia is **not a gendered language**.</x-trans>
<x-trans>Gender has **no effect** on grammatical rules—it is **purely semantic**, used only when relevant to specify the gender of an object or person.</x-trans>

<x-trans>Gender is marked by the prefixes **"vi-"** (male) and **"me-"** (female).</x-trans>
<x-trans>If gender **does not need to be specified**, the noun remains **unmarked**.</x-trans>

!!! note "<x-trans>Using Gender</x-trans>"

    <x-trans>By default, gender **should not be marked** unless it is **explicitly relevant**.</x-trans>
    <x-trans>For example, when referring to a child, **"filio"** is used unless specifying **"son"** (*vifilio*) or **"daughter"** (*mefilio*).</x-trans>

## **<x-trans>Examples</x-trans>**

- **<x-trans>Parent</x-trans>**

    - **<x-trans>Any gender</x-trans>:** pareno
    - **<x-trans>Father</x-trans>:** vipareno
    - **<x-trans>Mother</x-trans>:** mepareno

- **<x-trans>Child (offspring)</x-trans>**

    - **<x-trans>Any gender</x-trans>:** filio
    - **<x-trans>Son</x-trans>:** vifilio
    - **<x-trans>Daughter</x-trans>:** mefilio
